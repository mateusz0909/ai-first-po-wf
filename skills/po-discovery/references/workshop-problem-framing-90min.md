# 90-minute problem-framing workshop — agenda + facilitator script

Goal: leave with a shared **problem statement**, an **impact map**, and the **one
assumption to test first**. Client already has a problem; you're framing the real
one and de-risking. Cap ~8 people, one decider. Remote: pre-build the board.

Say this at the start (verbatim-ish):
> "By the end we'll agree on what problem we're really solving and the first
> thing we'll test — not solve it all today. I'll keep us on time and make sure
> everyone's heard. I'm neutral on the answer."

Ground rules (post them): one conversation at a time · write-first then share ·
disagree with ideas not people · parking lot for tangents · decider commits at
the end.

---

## 0:00–0:10 — Frame & context (10m)
- You: restate why we're here + the goal + agenda (2m).
- Round-robin: each person, one sentence — "the last time this problem bit us."
  (write-first 2m, share 6m). You capture verbatim pains on stickies.

## 0:10–0:30 — What's the real problem? (20m)
- **Write-first (5m, silent):** everyone writes "the problem is…" stickies.
- **Share + cluster (8m):** round-robin onto the board, affinity-cluster.
- **5 Whys on the top cluster (7m):** you ask "why is that a problem?" up to 5×.
  Tag each item: symptom / problem / already-a-solution.
- Output: a candidate problem, root-cause-deep.

## 0:30–0:40 — Draft the problem statement (10m)
- Fill the template live on the board:
  > "**[Who]** struggles to **[do what]** because **[root cause]**, which causes
  > **[impact/metric]**. Today they **[workaround]**. Success = **[measurable
  > change]**."
- Dot-vote if there are competing versions. Get the decider's nod. Park the
  proposed solutions — don't design yet.

## 0:40–1:05 — Impact map: connect problem → what to build (25m)
Build the spine left-to-right on the board:
- **Why (3m):** the goal/metric (from the problem statement).
- **Who (5m, write-first):** actors who can move that metric (users, buyer,
  support, ops…).
- **How (10m):** per actor, what behaviour change would help? (impacts)
- **What (7m):** only now — candidate deliverables per impact.
- Point out: the client's original request is ONE "what" branch. There are
  others. Don't commit yet.

## 1:05–1:20 — Assumption mapping + pick the risky one (15m)
- **Write-first (4m):** "for this to work, we're assuming…" stickies.
- **Place on a 2×2 (6m):** axes = how risky if wrong × how uncertain we are.
  Top-right = risky + uncertain = test first.
- **Dot-vote the top-right (3m).** Pick ONE assumption to test.
- Phrase it testable: "we believe [X]; we'll know we're wrong if [observable]."

## 1:20–1:30 — Close (10m)
- Read back: problem statement · the riskiest assumption · impact-map branch
  we're exploring.
- Owners + next step + date. "Next: we prototype this and you'll get to try it."
- Thank the room.

---

## If you have less time
- **60 min:** cut impact map to Why/Who/What (skip fine-grained How), 10m
  assumptions.
- **45 min:** problem statement (20m) + assumption 2×2 (20m) + close (5m). Skip
  impact map; do it async.

## Facilitator reminders
Write-first every generative step. Decider votes last/private. Timebox aloud.
Park tangents. Count to 7 after a question before rescuing it. See
`facilitator-playbook.md` for handling dominant voices, silence, and conflict.
